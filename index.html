MEAN stack orientation help

Congratulations on starting your MEAN stack journey! There are MongoDB, Express.js, Angular.js and Node.js...so far so good, but how to keep track of what they are and everything else. To start with, the order the names are presented in makes no sence to me. Therefor they will not be presented in MEAN order. I hope this page is going to help you cover some of the basics. (This guide will only cover comments around installation from a Mac users view.)

Node.js is a JavaScript run-time environment. Node.js allows you to write both server- and client-side JavaScript, and give us the possibility to and share code between client and server side (JavasScript end-to-end). Node.js can handle multiple web requests at the same time, they are all processed in the same thread (using a basic event model). NOde.js is open-source! One of Node.js strength is that it is fast to implement, both when it comes to installations and development.

Comment: Make sure to install and run a stable version (usually even version numbers) of Node.js when you develope. Node Package Manager (npm) is the Node.js default package manager. It lets you install custom written modules/packages from other developers (3rd party packages). See below for Usefull packages to use.

Connect is a HTTP server framework that allows you to use middleware (so basically a middleware framework). Learn more about Connect below under Usefull packages.

Node.js documentation

What it is: Node.js web application framework (higher level framework that more or less extends and built upon Connect). Express acts as a light layer on top of Node.js, making it easier to develop Node.js web applications. Provieds an easy-to-use API to interact with the webserver.

Comment: Middlewares are used to modify request objects and response objects. They can either end an HTTP request or pass it along to another middleware, like a chain, with help from the next object. One of the most important custom packages to install with npm is Express.

Nodemon

What: What, wrapper for Node.js. Allows you to keep your process up and running and automatically reloads it (restarts the server) for you when you save changes. This is perfect when you are developing and do not want to manually restart the server all the time. (If you use a start script, you can use 'npm run start dev' to run your code with nodemon, otherview you just replace node with nodemon in command line.)

Connect

Connect is a HTTP server framework that allows you to use middleware (so basically a middleware framework). Connect takes the build-in HTTP server functionality and extends it by adding a plugin framework. The plugins acts as middleware. Middleware are really functions and they live between different layers in your application, and works like a hidden connection.

Middleware and libraries for Connect and Express (link to more info)

  • bodyparser: Parse incomming request bodies in a middleware before the handlers. Can only handle singlepart (headers and body) body, not multipart bodies.
  • connect-flash: Used to store flash messages. Makes message available to the next page that is to be rendered.
  • cookieparser: Parse the cookie header to store data in browser when a session is eastablished on the server-side.
  • errorhandler: Helps handling errors in a development environment.
  • express-session: Used to identify users so that every user visiting the application will be assigned a unique session, so that a unique user state can be stored.
  • morgan: Simplifies the process of logging HTTP requests and errors to the application.
  • multer: Handles multipart bodies and form-data. I primarly used for uploading files.
  • passport: Provides you with a simple way to help you handle authentication. It authenticates requests.
  • serve-index: Serves pages that contain directory listings for a given path.
  • serve-static: Serves files from within given root directory. If a file is not found this module will move on to the next middleware instead of sending a 404 response.

dotenv

moment.js

JavaScript date library for parsing, validating, manipulating, and transforming dates. If you for example want to show date in other order or without time stamp.

You can for example access stored weather data via NASAs API. And then you can use that as input for your weather app. REST in itself is a software architectual style. Many APIs of today are of RESt type.

Postman is a API platform for building and using APIs. It comes as an application and you can use it to test your Express REST API
