The MEAN stack

The MEAN stack is a JavaScript-based framework for developing web applications. All components of the MEAN stack support programs that are written in JavaScript, therefore MEAN stack applications can be written in one language for both server-side and client-side execution environments.

Node is a JavaScript run-time environment. It allows us to write both server- and client-side JavaScript, and gives us the possibility to share code between client and server side (JavasScript end-to-end). Node can handle multiple web requests at the same time, they are all processed in the same thread (using a basic event model). One of Node's strengths is that it is fast to implement, both when it comes to installations and development.

Make sure to install and run a stable version of Node when you develop. Usually even version numbers are stable.

Node is open-source! It gives you the opportunity to install custom written modules/packages from other developers (3rd party packages). You'll use the Node Package Manager (npm) to install the packages. Please see below for Useful packages to use.

Connect is a HTTP server framework that allows you to use middleware. Learn more about Connect below under Useful packages.

Learn more in the Node.js documentation

One of the most important custom packages to install with npm is Express. It is a web application framework (higher level framework that more or less extends and built upon Connect). Express acts as a light layer on top of Node, making it easier to develop Node web applications. It provides an easy-to-use API to interact with the webserver.

Express also uses middleware. Middleware are used to modify request objects and response objects. They can either end an HTTP request or pass it along to another middleware, like a chain, with help from the next object.

Learn more at the home of Express.js

MongoDB is a document-oriented database (NoSQL). Unlike a relational database (SQL), it stores all information for a given object in a single instance in the database, and every stored object can be different from every other. This is suitable for a Node application and makes it a fast alternative for a database

Learn more in the MongoDB documentation

MongoDB Atlas is a cloud database service. In other words, this is where you can host your database and your data is stored here.

This is a separate service and you'll have to create an account via their webpage to use it.

Learn more in the MongoDB Atlas documentation

MongoDB Compass is the GUI (graphical user interface) for MongoDB. In other words, it is an interactive tool for querying, optimizing, and analyzing your data.

This is a separate application that you'll have to download and install to be able to run on your computer. You connect MongoDB Compass with your MongoDB Atlas. It is very visual and easy to use when working with your database.

Learn more in the MongoDB Compass documentation

Mongosh is the MongoDB shell. It is an environment for interacting with the MongoDB deployment. The previous MongoDB shell was called only ‘mongo’.

This is a command-line tool that can be installed from the MongoDB Atlas webpage (requires that you have Homebrew installed on your Mac). You connect your shell with your MongoDB Atlas.

Learn more in the Mongosh documentation

Mongoose is built on top of the MongoDB driver and it is an object modeling tool. In other words, it provides you a way to model your data.

You have to use 'npm install' to be able to use Mongoose on your computer.

Learn more in the Mongoose documentation

Angular is a client-side JavaScript framework. It was developed to make it easy to well-designed and structures webpages and applications. Angular extends HTML attributes with help from ng-directives, and binds data to HTML elements using expressions {{ }}. Nice-to-know is that 'ng' is a core module of Angular.

Angular uses TypeScript (which is a superset of JavaScript) and the code is transpiled into JavaScript. To work with Angular directly from your command-line you can useAngular CLI.

Learn more in the Angular.js documentation

Nodemon

Nodemon is a wrapper for Node. It allows you to keep your process up and running and automatically reloads it (restarts the server) for you when you save changes. This is perfect when you are developing and do not want to manually restart the server all the time. (When you have it installed and want to use it you just replace node with nodemon in the command line.)

Connect

Connect is a HTTP server framework that allows you to use middleware (so basically a middleware framework). Connect takes the built-in HTTP server functionality and extends it by adding a plugin framework. The plugins act as middleware. Middleware are really functions and they live between different layers in your application, and work like a hidden connection.

Middleware and libraries for Connect and Express (link to more info)

  • body-parser: Parse incomming request bodies in a middleware before the handlers. Can only handle singlepart (headers and body) body, not multipart bodies.
  • connect-flash: Used to store flash messages. Makes message available to the next page that is to be rendered.
  • cookieparser: Parse the cookie header to store data in browser when a session is eastablished on the server-side.
  • errorhandler: Helps handling errors in a development environment.
  • express-session: Used to identify users so that every user visiting the application will be assigned a unique session, so that a unique user state can be stored.
  • morgan: Simplifies the process of logging HTTP requests and errors to the application.
  • multer: Handles multipart bodies and form-data. I primarly used for uploading files.
  • passport: Provides you with a simple way to help you handle authentication. It authenticates requests.
  • serve-index: Serves pages that contain directory listings for a given path.
  • serve-static: Serves files from within given root directory. If a file is not found this module will move on to the next middleware instead of sending a 404 response.

Dotenv

Loads environment variables frmom a .env file. You can use this to store keys and values that you do not want to show in code. For example private keys when connecting to you Mongo database.

Moment.js

JavaScript date library for parsing, validating, manipulating, and transforming dates. If you for example want to show date in other order or without time stamp.

REST API = Representational State Transfer Application Programming Interface. One way to put it is that an API lets you access data from an external source. You can for example access stored weather data via NASAs API. And then you can use that as input for your weather app. REST in itself is a software architectural style. Many APIs of today are of REST type.

Postman is an API platform for building and using APIs. It comes as an application and you can use it to test your Express REST API

A template/view engine enables you to use static template files in your application. At runtime it transforms the template into an HTML file that is sent to the client/browser. The purpose is to make it easier to design an HTML page.

Examples: Pug/Jade, EJS, Handlebars, Mustache
